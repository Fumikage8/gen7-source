1. Introduction

This package is Zorro.
This package provides functions to sign and verify unique signatures with short overhead using OPSSR style signatures, with RSA, AES-CMC for the permutation function and SHA1 for the hash function.
The signatures are unique: for a given message and key, there is only one signature of it, that is why one should not use this package for key-exchange protocols.
The signature is public, even if it seems encrypted, one can extract the message using only public information.

This package may be used in the retail product.

2. Notes

The perl scripts sign.pl and verify.pl in the tools subdirectory implement this signature scheme. To use these scripts, installation of some depedencies are required. These dependencies are not packaged within this package and must be downloaded separately. Informations on how to do this easily are present in comments at the beginning of the signing script sign.pl.

6. Installation

Package Location

Extract this package to any directory.
Nintendo recommends that you put this package in a directory with a path that does not include spaces or double-byte characters.
After locating this package in the target directory, run setup.wsf to set the CTRMW_ZORRO_ROOT environment variable to the absolute path of the package's root directory.

Incorporating in Applications

Link to the following header files and libraries to include this middleware in an application.

Header File
include/mw/zorro/zorro.h
Libraries
library/(platform)/(target)/libmw_zorro.*.a (fast or small)
